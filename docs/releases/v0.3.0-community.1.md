# Mobazha Community Edition v0.3.0-community.1

Status: release candidate. This version has not been tagged or published.

## Highlights

- Self-hosted peer-to-peer marketplace node with store, listing, order, fulfillment, dispute, rating, notification, wallet, and administration workflows.
- Community payment runtime limited to BTC, BCH, and LTC.
- Backend-authoritative runtime capability projection for clients.
- Versioned payment-plugin architecture that keeps policy, verification, settlement gates, audit, and key custody in Core.
- Retained and sanitized public development history with original authorship and timestamps where approved.

## Compatibility

- Go 1.26.4 is required for the supported source build.
- The default build uses the pure-Go `goolm` implementation.
- The matching frontend release candidate is `mobazha-unified` v0.3.0-community.1.
- The Community Edition capability manifest is `config/editions/community.json`.

## Verification

Before tagging this candidate, release maintainers must run:

```bash
./scripts/community/audit-public-history.sh
./scripts/community/check-capabilities.sh
./scripts/community/check-vulnerabilities.sh
go vet -tags goolm ./...
go test -tags goolm -race -count=1 -timeout 10m ./...
go build -tags goolm -o /dev/null .
```

The final release must also publish checksums, provenance, a reviewed SBOM, third-party license material, and the exact source commit used for every artifact.

## Known limitations

- This release candidate is not a stable production release.
- ZEC is intentionally outside this candidate while its production monitoring and seller-settlement journey is completed.
- Additional payment providers are not bundled or enabled by this edition.
- Signed artifacts, installation packages, and reproducibility attestations are pending final release approval.

## Security

Do not report vulnerabilities in a public issue. Follow the private reporting process in [`SECURITY.md`](../../SECURITY.md).
