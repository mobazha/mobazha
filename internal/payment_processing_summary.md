# 支付和订单处理核心逻辑总结

## 1. 多币种钱包系统 (Multiwallet)

### 1.1 钱包初始化
- 支持多种加密货币：BTC、ETH、BCH、LTC、ZEC、SOL等
- 每种币种对应独立的钱包实现
- 支持主网和测试网环境
- 统一的数据库存储和日志管理

### 1.2 钱包功能
- 交易监听和订阅
- 余额查询
- 交易发送
- 地址管理
- 交易历史记录

## 2. 订单处理系统 (OrderProcessor)

### 2.1 核心组件
- 身份验证系统
- 数据库管理
- 消息传递系统
- 多币种钱包集成
- 事件总线
- 汇率提供者

### 2.2 订单状态管理
- 订单创建
- 支付处理
- 订单确认
- 订单履行
- 订单完成
- 纠纷处理

## 3. 支付流程

### 3.1 支付初始化
- 根据订单币种选择对应钱包
- 获取托管地址
- 计算支付金额
- 生成支付信息

### 3.2 支付执行
- 调用钱包的Spend()方法
- 发送资金到托管地址
- 等待交易确认
- 更新订单状态

### 3.3 支付确认
- 钱包监听托管地址交易
- 验证交易金额和地址
- 确认交易状态
- 触发订单状态更新
